## Overview

Adds a new SSH public key to your account. Once added, the key can be selected when provisioning new server instances to enable SSH access.

## Request Body

<ParamField body="name" type="string" required>
  A display name for the SSH key. This name is used when selecting keys during instance creation.
</ParamField>

<ParamField body="pubkey" type="string" required>
  The full SSH public key string. Supported formats include RSA, ED25519, and ECDSA keys.
</ParamField>

<RequestExample>

```bash cURL

  -H "Authorization: Bearer YOUR_API_TOKEN" \
```

## Response Fields

<ResponseField name="id" type="integer">
  Unique identifier for the newly created SSH key
</ResponseField>

<ResponseField name="fingerprint" type="string">
  The MD5 fingerprint of the SSH key in colon-separated hexadecimal format
</ResponseField>

<ResponseField name="pubkey" type="string">
  The full public key string that was uploaded
</ResponseField>

<ResponseField name="comment" type="string">
  The comment field extracted from the SSH key, typically containing `user@hostname`
</ResponseField>

<ResponseField name="name" type="string">
  The display name you provided for the SSH key
</ResponseField>

## Supported Key Types

| Type | Example Prefix |
|------|----------------|
| RSA | `ssh-rsa AAAA...` |
| ED25519 | `ssh-ed25519 AAAA...` |
| ECDSA | `ecdsa-sha2-nistp256 AAAA...` |

## Generating SSH Keys

If you need to generate a new SSH key pair, you can use the following commands:

```bash ED25519 (Recommended)
ssh-keygen -t ed25519 -C "your_email@example.com"
```

```bash RSA (4096-bit)
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

After generating, the public key (ending in `.pub`) can be uploaded using this endpoint.
