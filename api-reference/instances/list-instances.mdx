## Overview

Returns a list of all bare metal server instances associated with your account. You can optionally filter results by project using the `project` query parameter.

## Query Parameters

<ParamField query="project" type="string">
  Filter instances by project ID. Format: `PROJ-{short_uuid}` (e.g., `PROJ-8ad55d52`)
</ParamField>

<RequestExample>

```bash cURL
curl https://api.dallas-idc.com/v2/instance \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

```bash cURL (with project filter)
curl "https://api.dallas-idc.com/v2/instance?project=PROJ-8ad55d52" \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

```python Python
import requests

# List all instances
response = requests.get(
    "https://api.dallas-idc.com/v2/instance",
    headers={"Authorization": "Bearer YOUR_API_TOKEN"}
)
print(response.json())

# List instances in a specific project
response = requests.get(
    "https://api.dallas-idc.com/v2/instance",
    params={"project": "PROJ-8ad55d52"},
    headers={"Authorization": "Bearer YOUR_API_TOKEN"}
)
print(response.json())
```

## Response Fields

<ResponseField name="uuid" type="string">
  The full UUID of the instance (e.g., `a1b2c3d4-e5f6-7890-abcd-ef1234567890`)
</ResponseField>

<ResponseField name="short_uuid" type="string">
  The shortened UUID for the instance (e.g., `a1b2c3d4`)
</ResponseField>

<ResponseField name="name" type="string">
  The user-defined name for the instance
</ResponseField>

<ResponseField name="hostname" type="string">
  The fully qualified domain name (FQDN) assigned to the instance
</ResponseField>

<ResponseField name="status" type="string">
  The current status of the instance. Possible values include:
  - `Provisioning` - Server is being set up
  - `Online` - Server is running and accessible
  - `Offline` - Server is powered off
  - `Rebooting` - Server is restarting
  - `Suspended` - Server has been suspended
  - `Deleting` - Server is being deprovisioned
</ResponseField>

<ResponseField name="facility" type="string">
  The data center location code where the instance is deployed (e.g., `DFW1`, `LAX1`)
</ResponseField>

<ResponseField name="core" type="string">
  The server configuration ID representing the hardware specs (e.g., `bm1.e2234.1`)
</ResponseField>

<ResponseField name="image" type="string">
  The operating system image installed on the instance (e.g., `almalinux-8`, `ubuntu-22.04`)
</ResponseField>

<ResponseField name="os_disk" type="string">
  The disk device path where the operating system is installed (e.g., `/dev/nvme0n1`)
</ResponseField>

<ResponseField name="networks" type="array">
  List of network types attached to the instance. Possible values:
  - `public` - Public internet connectivity
  - `private` - Private network connectivity within the facility
</ResponseField>

<ResponseField name="ssh_keys" type="array">
  List of SSH key names that were added to the instance for authentication
</ResponseField>

<ResponseField name="tags" type="array">
  User-defined tags for organizing and filtering instances
</ResponseField>

<ResponseField name="metadata" type="array">
  Array of key-value pairs containing system-defined metadata
</ResponseField>

<ResponseField name="custom_metadata" type="object">
  User-defined key-value pairs for custom instance metadata
</ResponseField>

<ResponseField name="project_id" type="string">
  The project ID this instance belongs to (e.g., `PROJ-8ad55d52`)
</ResponseField>

<ResponseField name="item_type" type="string">
  The type of resource, always `instance` for this endpoint
</ResponseField>

<ResponseField name="create_date" type="string">
  The date and time when the instance was created (format: `YYYY-MM-DD HH:MM:SS`)
</ResponseField>

<ResponseField name="ip_addresses" type="array">
  List of IP addresses assigned to the instance
</ResponseField>

## IP Address Object

Each entry in the `ip_addresses` array contains:

| Field | Type | Description |
|-------|------|-------------|
| `address` | string | The IP address assigned to the instance |
| `type` | string | Network type: `public` or `private` |
| `gateway` | string | The gateway IP address for this network |
| `netmask` | string | The subnet mask (e.g., `255.255.255.0`) |

## Use Cases

- **Infrastructure Overview**: List all deployed servers across your account
- **Project Management**: Filter instances by project to view resources in a specific environment
- **Monitoring**: Check the status of all instances to identify any issues
- **Inventory Management**: Retrieve instance details for asset tracking and documentation
- **Automation**: Programmatically fetch instance information for scripts and workflows
