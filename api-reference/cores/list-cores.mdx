## Overview

Returns a list of all available server configurations (called "cores") that can be provisioned. Each core represents a specific hardware configuration including CPU, memory, storage, and network specifications along with hourly pricing.

<RequestExample>

```bash cURL
curl https://api.dallas-idc.com/v2/core \
  -H "Authorization: Bearer YOUR_API_TOKEN"
```

```python Python
import requests

response = requests.get(
    "https://api.dallas-idc.com/v2/core",
    headers={"Authorization": "Bearer YOUR_API_TOKEN"}
)
print(response.json())
```

```javascript JavaScript
const response = await fetch('https://api.dallas-idc.com/v2/core', {
```

## Response Fields

<ResponseField name="id" type="string">
  The unique identifier for the core configuration, used when creating instances (e.g., `bm1.e2234.1`, `bm1.epyc7402.1`)
</ResponseField>

<ResponseField name="name" type="string">
  Human-readable name of the server configuration, typically the CPU model
</ResponseField>

<ResponseField name="rate" type="number">
  The price per interval (hourly rate in USD)
</ResponseField>

<ResponseField name="interval" type="string">
  The billing interval for the rate (currently `hourly` for all cores)
</ResponseField>

<ResponseField name="description" type="string">
  Optional description of the server configuration
</ResponseField>

<ResponseField name="default_disk" type="string">
  The default disk device path for OS installation (e.g., `/dev/nvme0n1`, `/dev/sda`)
</ResponseField>

<ResponseField name="metadata" type="array">
  Array of key-value pairs containing hardware specifications
</ResponseField>

## Metadata Fields

Each core includes detailed hardware specifications in the `metadata` array:

| Key | Description | Example Values |
|-----|-------------|----------------|
| `cpus` | Number of physical CPUs | `1`, `2` |
| `cpu_model` | CPU model name | `Intel Xeon E-2234`, `AMD EPYC 7402` |
| `cpu_cores` | Number of cores per CPU | `4`, `6`, `12`, `16`, `24` |
| `memory` | Total RAM | `32GB`, `64GB`, `128GB`, `256GB` |
| `disk_size` | Storage capacity per disk | `512GB`, `960GB`, `1.92TB`, `3.84TB` |
| `disk_count` | Number of disks | `1`, `2`, `4` |
| `disk_type` | Storage technology | `NVMe`, `SSD`, `HDD` |
| `nic_speed` | Network interface speed | `1Gbps`, `10Gbps` |
| `nic_count` | Number of network interfaces | `2` |

## Use Cases

- **Instance Provisioning**: Select a core configuration when creating a new bare metal server instance
- **Capacity Planning**: Review available hardware configurations and pricing for infrastructure planning
- **Cost Estimation**: Calculate expected costs based on hourly rates before deployment
- **Hardware Selection**: Compare CPU, memory, and storage options to match workload requirements
