## Overview

Returns the current status and details of an asynchronous job. Jobs are created when you perform long-running operations such as provisioning a new server instance. Use this endpoint to monitor the progress of these operations.

## Path Parameters

<ParamField path="job_id" type="string" required>
  The unique job identifier. This is returned when you initiate an asynchronous operation like creating an instance.
</ParamField>

## Response Fields

<ResponseField name="uuid" type="string">
  The full UUID of the job (e.g., `abc12345-6789-0def-ghij-klmnopqrstuv`)
</ResponseField>

<ResponseField name="short_uuid" type="string">
  The shortened UUID for the job (e.g., `abc12345`)
</ResponseField>

<ResponseField name="status" type="string">
  The current status of the job. Possible values:
  - `pending` - Job is queued but not yet started
  - `running` - Job is currently in progress
  - `completed` - Job finished successfully
  - `failed` - Job encountered an error and stopped
  - `cancelled` - Job was cancelled by the user
</ResponseField>

<ResponseField name="type" type="string">
  The type of operation being performed. Possible values:
  - `provision` - Server provisioning
  - `deprovision` - Server termination
  - `reboot` - Server restart
  - `reinstall` - Operating system reinstallation
</ResponseField>

<ResponseField name="progress" type="integer">
  The completion percentage of the job (0-100)
</ResponseField>

<ResponseField name="message" type="string">
  A human-readable description of the current job step or final result
</ResponseField>

<ResponseField name="error" type="string">
  Error message if the job failed. Only present when `status` is `failed`
</ResponseField>

<ResponseField name="resource_type" type="string">
  The type of resource this job is operating on (e.g., `instance`)
</ResponseField>

<ResponseField name="resource_id" type="string | null">
  The ID of the resource being created or modified. May be `null` if the resource hasn't been created yet or if the job failed before creation.
</ResponseField>

<ResponseField name="create_date" type="string">
  The date and time when the job was created (format: `YYYY-MM-DD HH:MM:SS`)
</ResponseField>

<ResponseField name="started_at" type="string | null">
  The date and time when the job started executing, or `null` if still pending
</ResponseField>

<ResponseField name="completed_at" type="string | null">
  The date and time when the job completed, or `null` if still in progress
</ResponseField>

## Polling for Job Status

When provisioning a server, you'll receive a job ID in the response. Poll this endpoint periodically to check the status:

```python Python
import requests
import time

def wait_for_job(job_id, token, timeout=600, interval=10):
    """Poll job status until completion or timeout."""
    headers = {"Authorization": f"Bearer {token}"}
    start_time = time.time()

    while time.time() - start_time < timeout:
        response = requests.get(
            f"https://api.dallas-idc.com/v2/job/{job_id}",
            headers=headers
        )
        job = response.json()

        if job["status"] == "completed":
            print(f"Job completed! Resource ID: {job['resource_id']}")
            return job
        elif job["status"] == "failed":
            raise Exception(f"Job failed: {job.get('error', job['message'])}")

        print(f"Progress: {job['progress']}% - {job['message']}")
        time.sleep(interval)

    raise TimeoutError("Job did not complete within timeout period")
```

## Use Cases

- **Provisioning Monitoring**: Track the progress of server deployments
- **Error Handling**: Detect and respond to provisioning failures
- **Automation**: Build automated workflows that wait for operations to complete
- **Status Reporting**: Display provisioning progress in dashboards or CI/CD pipelines
