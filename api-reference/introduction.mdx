## Base URL

All API requests should be made to:

```
https://api.dallas-idc.com/v2
```

## Authentication

All endpoints require Bearer token authentication. Include your API token in the Authorization header:

```bash
Authorization: Bearer YOUR_API_TOKEN
```

See the [Authentication guide](/authentication) for details on obtaining a token.

## Response Format

All responses are returned in JSON format. Successful responses typically include the requested data, while error responses include an error message and details.

### Success Response

```json
{
  "data": {
    // Response data
  }
}
```

### Error Response

```json
{
  "error": "Error message",
  "message": "Detailed description"
}
```

## HTTP Status Codes

| Code | Description |
|------|-------------|
| 200 | Success |
| 201 | Created |
| 400 | Bad Request - Invalid parameters |
| 401 | Unauthorized - Invalid or missing token |
| 403 | Forbidden - Insufficient permissions |
| 404 | Not Found - Resource doesn't exist |
| 422 | Unprocessable Entity - Validation error |
| 429 | Too Many Requests - Rate limit exceeded |
| 500 | Internal Server Error |
