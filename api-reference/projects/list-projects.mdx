## Overview

Returns a list of all projects associated with your client account. Projects are used to organize and group related resources like instances. You can optionally include quota information to see resource limits and current usage.

## Query Parameters

<ParamField query="client_id" type="integer">
  Filter projects by client ID. If omitted, returns projects for the authenticated user's client account.
</ParamField>

<ParamField query="include_quotas" type="integer">
  Set to `1` to include quota information in the response. When enabled, each project will include resource limits and current usage statistics.
</ParamField>

## Response Fields

<ResponseField name="uuid" type="string">
  The full UUID of the project (e.g., `8ad55d52-c4a7-473a-8831-d20c49021538`)
</ResponseField>

<ResponseField name="short_uuid" type="string">
  The shortened UUID used in the project_id (e.g., `8ad55d52`)
</ResponseField>

<ResponseField name="project_id" type="string">
  The unique project identifier in the format `PROJ-{short_uuid}`
</ResponseField>

<ResponseField name="status" type="string">
  The current status of the project. Possible values include:
  - `Online` - Project is active and operational
  - `Suspended` - Project has been suspended
</ResponseField>

<ResponseField name="name" type="string | null">
  The user-defined name for the project, or `null` if not set
</ResponseField>

<ResponseField name="metadata" type="array">
  Array of key-value pairs containing custom project metadata
</ResponseField>

<ResponseField name="item_type" type="string">
  The type of resource, always `project` for this endpoint
</ResponseField>

<ResponseField name="create_date" type="string">
  The date and time when the project was created (format: `YYYY-MM-DD HH:MM:SS`)
</ResponseField>

<ResponseField name="client_id" type="integer">
  The unique identifier of the client account that owns this project
</ResponseField>

<ResponseField name="quotas" type="object">
  Resource quota information (only included when `include_quotas=1`). Contains:
  - `max_ips` - Maximum number of IP addresses allowed
  - `max_servers` - Maximum number of servers allowed (0 = unlimited)
  - `used_servers` - Current number of provisioned servers
  - `used_ips` - Current number of allocated IP addresses
</ResponseField>

<ResponseField name="unpaid_invoices" type="integer">
  The number of unpaid invoices associated with this project
</ResponseField>

## Quotas Object

When `include_quotas=1` is specified, each project includes a `quotas` object:

| Field | Type | Description |
|-------|------|-------------|
| `max_ips` | integer | Maximum number of IP addresses the project can use |
| `max_servers` | integer | Maximum number of servers allowed (0 = unlimited) |
| `used_servers` | integer | Number of servers currently provisioned |
| `used_ips` | integer | Number of IP addresses currently in use |

## Use Cases

- **Account Overview**: List all projects to get a high-level view of your infrastructure organization
- **Capacity Planning**: Include quotas to understand resource limits and current usage
- **Billing Review**: Check for unpaid invoices across all projects
- **Multi-Project Management**: Retrieve all projects when managing multiple environments or clients
- **Automation**: Programmatically enumerate projects for infrastructure-as-code workflows
