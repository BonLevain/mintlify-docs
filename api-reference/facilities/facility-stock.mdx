## Overview

Returns real-time server availability for a specified datacenter facility. Servers are categorized by their availability status: available for immediate deployment, low stock, or unavailable.

<ParamField path="location" type="string" required>
  The facility code (e.g., DFW1, LAX1, SLC1). See [List Facilities](/api-reference/facilities/list-facilities) for available locations.
</ParamField>

## Response Fields

<ResponseField name="facility" type="string">
  The facility code that was queried
</ResponseField>

<ResponseField name="available" type="array">
  List of server core types that are immediately available for deployment
</ResponseField>

<ResponseField name="low" type="array">
  List of server core types with limited availability (order soon)
</ResponseField>

<ResponseField name="unavailable" type="array">
  List of server core types that are currently out of stock
</ResponseField>

## Use Cases

- **Pre-deployment Check**: Verify server availability before creating an instance
- **Capacity Planning**: Monitor stock levels across facilities for provisioning decisions
- **Multi-Region Deployment**: Find facilities with available capacity for your required server type
- **Inventory Monitoring**: Track stock levels for automated procurement workflows

## Stock Status Guide

| Status | Meaning | Recommendation |
|--------|---------|----------------|
| `available` | Ready for immediate deployment | Safe to provision |
| `low` | Limited quantity remaining | Order soon if needed |
| `unavailable` | Currently out of stock | Check other facilities or wait for restock |
